## 技术方案评审

### 1. 前端架构

#### 技术选型
- **框架**：React 18 + TypeScript
- **状态管理**：Zustand
- **UI组件**：Ant Design 5.0
- **表单方案**：React Hook Form + Zod
- **构建工具**：Vite

#### 目录结构建议
```
src/
├── components/
│   ├── common/       # 通用组件
│   ├── workflow/     # 流程组件
│   └── kanban/       # 看板组件
├── pages/
│   ├── workbench/    # 工作台
│   ├── kanban/       # 看板
│   └── detail/       # 详情页
├── hooks/            # 自定义Hooks
├── services/         # API服务
├── stores/           # 状态管理
└── types/            # TypeScript类型
```

### 2. 后端架构

#### 技术选型
- **框架**：Node.js + Express/NestJS
- **数据库**：PostgreSQL + MongoDB
- **ORM**：Prisma
- **文件存储**：本地/OSS

#### 核心API设计

| 模块 | API | 方法 | 说明 |
|------|-----|------|------|
| 班车 | /api/shuttle | GET/POST | 班车列表/创建 |
| 应用 | /api/application | GET/POST | 应用列表/添加 |
| 节点 | /api/node/:id | GET/PUT | 节点详情/更新 |
| 物料 | /api/material | GET/POST/PUT | 物料 CRUD |
| 审核 | /api/review | POST | 提交审核 |
| 看板 | /api/kanban | GET | 看板数据 |

### 3. 数据库设计

#### 核心表结构

**shuttle（班车表）**
- id, name, type, status, applicant, apply_time, created_at, updated_at

**application（应用表）**
- id, shuttle_id, app_name, package_name, app_type, version, current_node, status, created_at

**node_config（节点配置表）**
- id, application_id, node_type, assignee, collaborators, status, reject_reason

**material（物料表）**
- id, application_id, language, app_name, description, keywords, icon, screenshots, is_gp, gp_link

**operation_log（操作日志表）**
- id, application_id, node_type, operator, action, details, created_at
