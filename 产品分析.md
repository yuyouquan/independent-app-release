## 一、产品概述

独立三方应用发布系统是一个用于管理独立应用发布流程的企业级工作流平台，主要服务于移动应用发布团队，实现从申请到上架的全流程管理。

### 1.1 产品定位
- **目标用户**：应用发布运营人员、产品经理、审核人员
- **核心价值**：规范化发布流程、追踪发布状态、提高发布效率

### 1.2 系统架构
```
┌─────────────────────────────────────────────┐
│              独立应用发布系统                │
├──────────────────┬──────────────────────────┤
│     工作台       │         看板            │
├──────────────────┼──────────────────────────┤
│ 申请列表 | 代办  │ 班车视角 | 产品视角 | 状态视角 │
└──────────────────┴──────────────────────────┘
```

## 三、核心业务流程

### 3.1 班车创建流程
```
选择班车类型 → 校验是否存在 → 生成班车名称 → 创建流程单
```

### 3.2 应用发布流程
```
通道发布申请 → 通道发布审核 → 物料上传 → 物料审核 → 应用上架 → 业务内测 → 灰度监控
```

### 3.3 审核回退流程
```
审核不通过 → 填写拒绝原因 → 通知责任人 → 回退到对应节点 → 显示拒绝状态和原因
```

## 四、数据模型设计要点

### 4.1 核心实体

| 实体 | 说明 |
|------|------|
| 班车 | 独立三方应用发布流程的容器 |

## 五、技术实现建议

### 5.1 前端技术栈
- React + TypeScript
- 状态管理：Zustand/Redux
- UI组件：Ant Design或自研
- 表单方案：React Hook Form + Zod
- 富文本：支持多语言切换

### 5.2 后端技术栈
- Node.js/Python
- 数据库：PostgreSQL（流程数据）+ MongoDB（物料数据）
- 文件存储：OSS/S3
- 消息通知：飞书Webhook

### 5.3 关键组件
- 多语言Tab切换组件
- 级联选择器（国家/品牌/机型）
- 图片上传组件（预览、尺寸校验）
- 流程状态可视化组件
- 审核会签组件
