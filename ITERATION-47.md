## 三、流程节点核对 ✅

### 3.1 节点流程图
```
通道发布申请 → 通道发布审核 → 物料上传 → 物料审核 → 应用上架 → 业务内测 → 灰度监控
```

### 3.2 节点扭转逻辑

| 当前节点 | 审核通过 | 审核拒绝 |
|----------|----------|----------|
| 通道发布申请 | → 通道发布审核 | - |
| 通道发布审核 | → 物料上传 | → 回退到通道发布申请 (显示拒绝原因) |
| 物料上传 | → 物料审核 | - |
| 物料审核 | 运营通过 → 老板审核<br>老板通过 → 应用上架 | → 回退到物料上传 (显示拒绝原因) |
| 应用上架 | - | 第三方平台数据只读 |
| 业务内测 | - | 收到拒绝 → 回退到物料上传 |
| 灰度监控 | - | 第三方平台数据只读 |

### 3.3 状态颜色

| 状态 | 颜色 | 说明 |
|------|------|------|
| 进行中 | 蓝色 🔵 | 有节点正在进行 |
| 已完成 | 绿色 🟢 | 所有节点完成 |
| 失败 | 红色 🔴 | 有节点拒绝 |
